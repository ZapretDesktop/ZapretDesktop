# Руководство по участию в разработке ZapretDeskop

Спасибо за интерес к проекту. Ниже — как сообщать об ошибках, предлагать улучшения и присылать изменения в код.

## Сообщение об ошибках и предложениях

- **Баг:** используйте [шаблон отчёта об ошибке](.github/ISSUE_TEMPLATE/bug_report.md) (Issue → Bug report). Укажите версию ZapretDeskop, ОС, шаги воспроизведения и ожидаемое поведение.
- **Идея / доработка:** создайте Issue с меткой `enhancement` или обсудите в существующем обсуждении.

## Требования к окружению для разработки

- **ОС:** Windows 10/11 (x64)
- **Python:** 3.10 или выше
- **Зависимости:** см. `requirements.txt` (PyQt6, psutil, requests и др.)

Установка зависимостей:

```bash
pip install -r requirements.txt
```

## Сборка и запуск из исходников

- Точка входа: `ZapretDeskop.py` или модуль, указанный в документации репозитория.
- Сборка автономного exe (PyInstaller): выполните `build.bat` в корне проекта (требуется Python и установленные зависимости).

Перед отправкой изменений убедитесь, что приложение запускается и основные сценарии (выбор стратегии, запуск/остановка, настройки) работают.

## Структура репозитория

- `src/core/` — ядро: конфиг, пути, обновления, генерация .bat, управление winws
- `src/ui/` — главное окно, системный трей, базовые окна и диалоги
- `src/dialogs/` — диалоги настроек, дополнений, редакторов и т.п.
- `src/editor/` — объединённый редактор (lists, drivers\etc, стратегии)
- `src/widgets/` — кастомные виджеты (меню, комбобокс, скроллбары и др.)
- `winws/` — среда выполнения zapret/Flowseal (bin, lists, .bat-стратегии)

Подробнее см. в [README.md](README.md).

## Pull Request

1. Создайте ответвление от актуальной ветки (обычно `main`).
2. Вносите изменения в логичных коммитах; по возможности привязывайте к существующим Issue.
3. В описании PR кратко укажите, что изменено и зачем; при необходимости — шаги проверки.
4. Убедитесь, что код не ломает запуск и базовый функционал.

Правки будут рассмотрены сопровождающими; возможны правки по замечаниям ревью.

## Код и стиль

- Стиль кода — в духе существующего проекта (Python, PyQt6).
- Избегайте жёстко зашитых путей и конфигураций; используйте существующие утилиты из `src/core/` (например, пути, конфиг).
- Новый функционал, затрагивающий безопасность или запуск внешних процессов, должен следовать уже заложенным проверкам (например, разрешённые .bat только из папки winws).

- Контакт сопровождающих: **ZapretDeskop@proton.me** (в т.ч. по поводу кодекса поведения и безопасности — см. [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md) и [SECURITY.md](SECURITY.md)).
